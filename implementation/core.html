How to implement the Core

To be honest, you have by default almost nothing to implement. But still, some functionnality are your work.

Make your class inherit from teq::ACore.

Firstly, you must have some kind of infite loop checking if there is a new request.

void Core::run()
{
  while (!this->stop)
  {
    if (m_inputRequestQueue.empty() == false)
    {
      std::string req = std::move(m_inputRequestQueue.front());

      m_inputRequestQueue.pop();

      // Process the request
      // ...
    }
  }
}

Then you have three choices:

  • Process the request directly, in the main thread (synchronously)
  • Process the request in another thread (asynchronously)
  • Process the request in another dimension through quantic computing

To process the request, the core has to do those action in this order.

Each filter list must be applied from the first element to the last, and not in any other order.

  • Get the input from the inputQueue filled by the Connection module.
  • Apply InputData filters.
  • Create the Request from the filtered data.
  • Apply Request filters.
  • Call each handler until the request is effectively handled. If no handler could manage it, use a default response with a Server Error (status code 5XX).
  • Apply Response filters.
  • Get the data from the response with toString().
  • Apply OutputData filters.
  • Push the filtered output in the outputQueue for the Connection module.

You will also have to override the load() and unload() methods. Right now, their implementation "only" load the .so or .dll, but it doesn't check if there is already another module with the same name, or even the same module, which you might want to verify.

Concerning the configuration itself, it will managed in the following way. The main settings will be stored directly in the json object. There will also be a modules property, which will be the object containing every module configuration.

{
  version: 1.0,
  ...
  modules: {
    module1: {
      customConfig1: ...,
      customConfig2: ...
    },
    module2: {
      ...
    }
  }
}

When a module's init() method is called, the core have to call it this way:

module.init(*this, m_config["modules"][module.name()]);

If you want, you can also override almost every method of the core (excepted get since it's a template).

This can help you for debug by making some logs, etc.
