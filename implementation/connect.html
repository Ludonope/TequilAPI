How to implement the Connection module ?

First, you need to implement module.

If you haven't done it yet, learn how to implement a module.

Make your class inherit from teq::IConnection.

First, you will have to implement the init() method.

void Connection::init(teq::ACore &core, nlohmann::json const &config)
{
  m_config = &config;
  m_core = &core;
  core.setConnection(this);
}

You then have to implement those

virtual void setInput(std::queue<http::Message> &queue) = 0;
virtual void setOutput(std::queue<http::Message> &queue) = 0;

Please take note to avoid confusion:

  • The module receive request from client and push them in the input queue (the setInput() one)
  • The core receive response from handler and push them in the output queue (the setOutput() one)

I know, using a std::queue is not thread safe. It will be replaced by a SafeQueue very soon. For now, let's do like if the std::queue was actually thread safe.

Now, since the teq::IConnection inherit from teq::IMainModule, you will have to implement those two

virtual void enable() = 0;
virtual void disable() = 0;

It will allow us to launch and stop a thread to manage network I/O asynchronously.

// Connection.hpp
class Connection : public teq::IConnection
{
public:
  // ...
private:
  // ...
  std::atomic<bool> m_stop;
  std::thread m_thread;
};

// Connection.cpp
void Connection::manageNetwork()
{
  while (!m_stop)
  {
    // Network I/O ...
  }
}

void Connection::enable()
{
  m_stop = false;
  m_thread = std::thread([this]() {
    this->manageNetwork();
  });
}

void Connection::disable()
{
  m_stop = true;
  m_thread.join();
}
